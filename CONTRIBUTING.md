# Contributing to Jules MCP

First off, thank you for considering contributing to Jules MCP! It's people like you who make the open source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs

- **Check the Issues**: Before opening a new issue, check if the bug has already been reported.
- **Use the Template**: If you find a new bug, please use the Bug Report template to provide as much information as possible.

### Suggesting Enhancements

- **Open an Issue**: Use the Feature Request template to describe the enhancement you'd like to see.
- **Explain the Why**: Tell us why this feature would be useful and how it should work.

### Pull Requests

1. **Fork the repository**.
2. **Create a new branch**: `git checkout -b feature/my-new-feature`.
3. **Make your changes**.
4. **Follow the code style**: Ensure your code follows the existing style and is well-documented.
5. **Write tests**: If applicable, add tests for your changes.
6. **Commit your changes**: Use descriptive commit messages.
7. **Push to your fork**: `git push origin feature/my-new-feature`.
8. **Open a Pull Request**: Provide a clear description of your changes and why they are needed.

## Code of Conduct

Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.

## Development Setup

1. Clone the repository.
2. Install dependencies: `npm install`.
3. Set up environment variables: Copy `.env.example` to `.env` and fill in the values.
4. Run in dev mode: `npm run dev`.

## Questions?

If you have any questions, feel free to open an issue or reach out to the maintainers.
